GET get_device
Получение/контроль подключения usb

Входные параметры:
нет

Выходные параметры:
res - null если нет или путь до папки usb 

----------------------------------------------
GET get_file_tree
Получить дерево файлов с usb

Входные параметры:
нет

Выходные параметры:
res - null если нет или словарь с деревом обектов (файлы и/или папки), где
name - имя папки/файла
type - тип (возможные значения directory - папка, file - файл)
children - массив дочерних объектов, если родительский является папкой

Пример:
{
    "res": {
        "name": "5861-7C47",
        "type": "directory",
        "children": [
            {
                "name": "test.pdf",
                "type": "file"
            },
            {
                "name": "B45MH4_7.40",
                "type": "file"
            },
            {
                "name": "B45MH4_4.90",
                "type": "file"
            },
            {
                "name": "test",
                "type": "directory",
                "children": []
            }
        ]
    }
}


----------------------------------------------
POST create_session_print
Создание сессии печати (папки)

Входные параметры:
нет

Выходные параметры:
id_session - ид сессии,
"params": [  - массив параметров
        {
            "operation": {
                "name": "print-two-side" - наименование действия
            },
            "price": стоимость двусторонней печати (1 шт.)
        }
]
* print-two-side - двусторонняя печать, print-one-side - односторонняя печать

----------------------------------------------
POST preview_file
Конвертация файла в pdf, возврат имени файла для предпросмотра

Входные параметры:
id_session - ид сессии
file_path - путь до файла (например, /media/bots/5861-7C47/Шаблон сервисного договора.docx)

Выходные параметры:
res - имя файла предпросмотра в формате pdf

----------------------------------------------
POST print_file
Печать файла

Входные параметры:
id_session - ид сессии
file_name - имя файла .pdf из папки(сессии) пользователя (формируется в методе preview_file. Например, Шаблон сервисного договора.pdf)
page_list - массив страниц печати (например, [1, 3, 6]). Необязательный параметр, по умолчанию печать всех страниц.
type_print - тип печати. Возможные значения 'two' - двусторонняя печать и 'one' - односторонняя печать. Необязательный параметр, по умолчанию 'one'
count_copy - количество копий (целое число). Необязательный параметр, по умолчанию 1.

Выходные параметры:
res - результат печати (строка)
